YieldSage v2 Proposal

From personal Telegram bot to Solana Risk Intelligence Platform. Always-on monitoring, team alerts, visual dashboard, comprehensive data.

Solana Focused Modal Serverless Next.js Dashboard

What We Have Today (v1)

YieldSage v1 is a Telegram bot running on a local machine. It monitors Solana DeFi yields, tracks stablecoin prices, and sends alerts. Built over a weekend, iterated over 3 days.

Data Sources (9)

  • DeFiLlama - 2700 pools + lending rates + stablecoin mcap + protocol TVL/fees/volume (7 endpoints)
  • Jupiter Price API v3 - 10 Solana stablecoin prices (30s)
  • CoinGecko - fallback price source (11 stablecoins)
  • Solana-yields scraper - 48 curated strategy APYs
  • TwitterAPI.io - 16 security/protocol accounts
  • Crypto News API - security + regulatory news
  • Helius - wallet balances
  • AllezLabs ClickHouse - Kamino transaction data
  • Dune SIM - vault outflow monitoring

Alert Types (9)

  • Stablecoin depeg (0.5% warning, 2% critical)
  • Lending utilization spike (>90%)
  • Stablecoin market cap drop (>5%)
  • Yield changes (TVL >$5M pools)
  • TVL drops (>20%, TVL >$1M)
  • Fee sustainability (0 revenue + high APY)
  • Stale data detection (per source)
  • Twitter security signals
  • News: exploit/hack detection

Bot Commands (10)

  • /top - Top yields with LLM risk analysis + utilization %
  • /strategies - 48 curated with risk flags
  • /protocol - Protocol health (TVL, age, fees, lending)
  • /kamino - Kamino lending from ClickHouse
  • /propose - AI yield allocation
  • /portfolio - Wallet positions
  • /alerts - Configure thresholds
  • /status - Source health
  • /sources - Monitored accounts

Storage

  • SQLite - local cache (pools, tweets, alerts, config)
  • ClickHouse - 8 tables for long-term analytics (APY snapshots, prices, lending rates, protocol TVL/fees, alerts)
  • 48 curated strategies with qualitative risk data

What's Missing

Current Limitations

  • Runs on a laptop (dies when lid closes)
  • Telegram only (no team visibility)
  • No visual dashboard (charts, trends)
  • Config via .env file (not user-friendly)
  • No freeze/blacklist event monitoring
  • No multisig governance tracking
  • No mint/burn event detection
  • No cross-signal correlation (Tier 1 incomplete)
  • No historical incident database

v2 Target

  • Always-on (Modal serverless, 24/7)
  • Slack alerts for the team + Telegram for personal
  • Next.js dashboard with charts, protocol pages, alert history
  • Web UI for configuration
  • USDC/USDT freeze tracking (on-chain events)
  • Multisig monitoring (from solmultisig.com data)
  • Stablecoin mint/burn events
  • Correlator engine (multi-signal threat scoring)
  • Incident history (Drift, Resolv, etc.)

Dashboard Pages

Next.js frontend deployed on Vercel, querying the Modal API + ClickHouse directly.

Overview

Real-time status of all monitored data. Source health indicators. Recent alerts timeline. Key metrics: total TVL tracked, stablecoin health, active alerts count.

Stablecoin Monitor

All monitored stablecoins with live price, deviation from peg, market cap trend, 7d sparkline chart. Freeze/blacklist event log. Mint/burn activity. Inspired by Pharos' peg tracking.

Protocol Health

Per-protocol page: TVL chart (30d), fees/revenue trend, lending markets with utilization heatmap, multisig structure (from solmultisig), Lindy score, risk assessment.

Yield Explorer

Top pools by APY with risk zones (safe/caution/high/extreme). LLM risk comments. Filter by category, risk level, protocol. APY history charts per pool.

Curated Strategies

The 48 curated stablecoin strategies with live APY, risk data, protocol health flags, yield sustainability flags. Compare mode. Links to deposit pages.

Alert History

Full timeline of all alerts with filters by type, severity, protocol. Exportable. Charts: alerts per day, by type, by protocol. Alert configuration UI.

Multisig Governance

All tracked Solana protocol multisigs. Current threshold, member count, roles. Change history (threshold changes, new members). Red flags on suspicious activity.

Settings

Alert thresholds (depeg, utilization, TVL drop, etc.). Monitored protocols. Notification preferences (Slack/Telegram). API keys management.

Implementation Roadmap

Incremental delivery. Each phase is usable independently.

Phase 1

Modal Deployment + Slack

1-2 days

Move YieldSage engine to Modal. Replace SQLite with ClickHouse for all state. Add Slack webhook for team alerts. The bot runs 24/7 without a laptop.

  • Wrap all jobs in @modal.function with modal.Cron
  • Secrets via modal.Secret
  • Slack webhook dispatcher (severity filtering: only warning/critical)
  • Keep Telegram as personal fallback
Phase 2

REST API

1-2 days

Expose YieldSage data via FastAPI on Modal. This is what the dashboard will consume.

  • /api/status - source health
  • /api/pools - top pools with risk data
  • /api/strategies - 48 curated enriched
  • /api/protocol/{name} - protocol health
  • /api/stablecoins - prices + mcap + depeg status
  • /api/alerts - history from ClickHouse
  • /api/charts/{type}/{id} - time-series for charts
  • /api/config - read/write alert thresholds
Phase 3

Dashboard MVP

2-3 days

Next.js frontend on Vercel. Overview + Stablecoin Monitor + Protocol Health + Alert History. Clean, dark UI.

  • Next.js 14+ with App Router
  • Tailwind + shadcn/ui
  • TanStack Query for data fetching
  • Recharts for time-series
  • 4 pages: Overview, Stablecoins, Protocols, Alerts
Phase 4

New Data Sources

3-5 days

Add the high-value sources we're missing.

  • Solmultisig data - scrape/API for 16 protocol multisig structures
  • CEX prices - Binance/Kraken websocket or REST for real market prices
  • Freeze/blacklist events - Helius webhook on USDC/USDT program for freeze instructions
  • Mint/burn events - Helius webhook on stablecoin mint authorities
  • Pyth oracle health - confidence intervals, staleness detection
  • APY history backfill - DeFiLlama /chart/{pool_id} for curator tracking
Phase 5

Advanced Features

Ongoing

The features that require the data foundation to be solid first.

  • Correlator engine - cross-signal threat scoring (depeg + TVL drop + tweet = critical)
  • Curator performance - Sharpe ratio, APY volatility, reaction time per curator
  • Automated incident reports - LLM-generated reports on critical events
  • Historical pattern detection - match incoming signals to known exploit patterns
  • Risk Score framework - 6-dimension scoring (Lindyness, dependency, liquidity, oracle, audit, yield source)
