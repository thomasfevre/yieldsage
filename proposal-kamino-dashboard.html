1. Problem Statement

Fragmented

10+ separate Hex dashboards for Kamino: fees, rates, liquidations, vault aggregates, position simulator, dollar rates, market metrics... No single source of truth.

Passive

Hex notebooks need manual refresh (15-30 min cache). Not designed for continuous monitoring during the day. You go to Hex to analyze, not to monitor.

Missing Stablecoin Risk

None of the Hex dashboards cover peg stability, blacklist events, reserve backing, or multi-source price consensus for stablecoins in Kamino vaults.

Not Operational

Designed for deep analysis, not for "what's happening right now?" Answering simple questions (is utilization near cap? any depegs?) requires opening multiple dashboards.

2. Solution

Build a "control room" for Kamino that consolidates live operations data into a single dashboard, adds a Pharos-powered stablecoin risk layer, and serves as the go-to tool for continuous monitoring.

What it IS

  • Live operations dashboard (30-60s refresh)
  • Stablecoin risk intelligence (Pharos + multi-source)
  • All Kamino markets, vaults, positions in one place
  • Activity feed (like Solscan, but Kamino-specific)

What it is NOT

  • Not replacing Hex (Hex stays for deep analytics)
  • Not a public product (internal Allez tool)
  • Not covering Venus yet (separate existing dashboard)
  • Not a position simulator (Phase 2)

Key Principles

  • Follow established patterns: lives in the existing venus-dashboard monorepo as apps/kamino, reuses shared packages
  • Same stack as Venus/Hyperdrive: React + Vite + Tremor + Express + Modal
  • Complement Hex: Hex for deep analysis, dashboard for live operations
  • Kamino-first: scoped to Kamino on Solana, stablecoins used in Kamino vaults

6. API Design

Markets

GET /api/markets - all markets + reserves
GET /api/markets/:name - single market
GET /api/markets/:name/history - 24h history

Vaults

GET /api/vaults - all vaults + allocations
GET /api/vaults/:name - single vault detail
GET /api/vaults/:name/history - 24h allocation changes

Positions

GET /api/positions/at-risk - near-liquidation
GET /api/positions/whales - top positions
?market=&drawdown_pct=10

Activity

GET /api/activity/feed - transaction feed
GET /api/activity/liquidations - liquidations
?hours=24&min_usd=10000&type=all

Stablecoin Risk

GET /api/stablecoins - all stables overview
GET /api/stablecoins/:symbol - full detail
GET /api/stablecoins/events - depeg/blacklist events

Alerts

GET /api/alerts - aggregated events
Cap breaches, depeg warnings, large txs, liquidation spikes
Computed server-side from thresholds

7. Phases & Scope

Phase 1 (This PRD)

  • Markets tab (reserves, utilization, caps, APY)
  • Vaults tab (allocations, drift, Allez vaults highlighted)
  • Positions tab (at-risk, whales, drawdown simulation)
  • Stablecoin Risk tab (Pharos + multi-source consensus)
  • Activity feed (transactions, liquidations)
  • Auto-refresh (30-60s)
  • Modal deployment (kamino.allez.xyz)

Phase 2 (Later)

  • Position simulator (pykamino backtest integration)
  • Push notifications (Slack/Telegram)
  • Authentication (if needed for external access)
  • Historical analytics beyond 24h
  • Mobile responsive layout

Out of Scope

  • Venus BNB Chain (already has its own dashboard)
  • Replacing Hex dashboards (they stay for deep analytics)
  • Public-facing product (internal Allez tool)
  • New ClickHouse data ingestion (all tables already exist)
